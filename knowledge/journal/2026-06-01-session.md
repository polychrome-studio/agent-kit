## Research mode gets web reach (M4 follow-up)
Tucker tested M4 routing (model switching works) but flagged that **research mode only
synthesizes the vault — it doesn't reach outside or offer to.** Correct: Amber had exactly
one knowledge source (vault + model weights), no external tools. So research could only
report gaps, not fill them.

Fix: enabled OpenRouter's built-in web search on research mode via the `:online` model
suffix → `anthropic/claude-opus-4.8:online`. Rewrote the research persona to use vault + web
together and "go find what's missing" rather than report the gap; cite notes vs URLs vs
general knowledge distinctly. `prettyModel()` strips `:online` so the label stays "opus 4.8".

### Decisions / why
- **`:online` over the `web` plugin** — they're the same feature; `:online` is the plugin with
  defaults. Tucker's call; plugin form only matters if we want to cap `max_results` for cost.
- **Cost** is a non-issue: ~2¢/search (Exa, ~5 results), minor vs Opus token costs, and not
  substitutable (search fetches what the model doesn't know).
- **Only research mode is `:online`, NOT companion/quick.** Key nuance surfaced to Tucker:
  `:online` FORCES a search on every request — it is *not* "the model goes online if needed."
  Forcing it on companion would tax every casual message (~1–2s + 2¢) for no gain and kill the
  snap. The real "online when needed, any mode" is **model-decided tool-use** — parked as the
  next capability (see [[roadmap]]).
